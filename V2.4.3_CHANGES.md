### 2. Feature Additions

#### Cue Editing Functionality
**Files Modified:** `public/app.js`

**Added:**
- `editingCueIndex` variable to track edit state
- `editExistingCue(index)` function to load cue data into editor
- "Edit" button in cue stack builder (custom cues only)
- Edit button handler that calls `editExistingCue()`
- Updated `saveCustomCueBtn` handler to support update mode
- Updated `closeCustomModal()` to reset editing state

**How it works:**
1. User clicks "Edit" on a custom cue
2. `editExistingCue()` loads cue data into the custom cue modal
3. User makes changes
4. Save button updates existing cue instead of creating new one
5. `editingCueIndex` is reset on close

**Lines modified:**
- Added edit button HTML in `renderCueStackBuilder()` (~line 2775)
- Added edit button event handlers (~line 2800)
- Added `editExistingCue()` function (~line 2945)
